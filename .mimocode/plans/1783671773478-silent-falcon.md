# Plan: Production Ready — C-AES Penalty Shootout

## Status: Current State

Proyek sudah **compiles clean** (0 warnings, `-Wall -Wextra -Wpedantic`), semua fitur PRD sudah jalan, arsitektur AES sudah diterapkan dengan barrel-header approach (pragmatis untuk C). Yang perlu diperbaiki adalah pelanggaran AES rules dan beberapa gap kecil.

---

## Issues yang Ditemukan

### 1. Entry File Salah Nama (AES101 — HIGH)
- **File:** `src/cli_main_entry.c`
- **Masalah:** Tidak punya prefix `root_` — violations naming convention AES101
- **Fix:** Rename ke `src/root_cli_main_entry.c`, update Makefile

### 2. Capabilities Include stdio.h (AES201 — CRITICAL)
- **File:** `src/registration/capabilities_registration_appender.c`
- **Masalah:** `#include <stdio.h>` di layer capabilities. Capabilities hanya boleh import `taxonomy_` dan `contract_`. Penggunaan `snprintf` adalah I/O system call.
- **Fix:** Ganti `snprintf` dengan `strncpy` + manual null-terminate. Hapus `#include <stdio.h>`.

### 3. CI-Equal Duplication (AES305 — MEDIUM)
- **File:** `capabilities_registration_validator.c`, `surfaces_registration_command.c`, `capabilities_search_resolver.c`
- **Masalah:** Fungsi `ci_equal` diduplikasi di 3 file berbeda
- **Fix:** Pindahkan ke `shared/` sebagai shared utility

### 4. Makefile Tidak Ada Target Spesifik
- **Masalah:** Tidak ada target untuk build individual file atau check
- **Fix:** Tambahkan target `check` untuk validasi struktur
