# Saran UI/UX #3 — Onboarding, Bantuan & Aksesibilitas

**Target:** Pengguna baru dan kejelasan aturan.

## 3.1 Layar "Cara Bermain" sebelum menu
Pengguna baru tidak tahu aturan (zona = poin, 7 tendangan, 5-7 peserta).
- **Valid:** YA — belum ada.
- **Tindakan:** Tambahkan entri "?" di splash atau tombol 'h' di menu utama yang
  membuka layar bantuan singkat (aturan + navigasi). Bisa pakai `tui_box` + teks.

## 3.2 Petunjuk konteks di tiap layar (footer sudah ada, perlu per layar)
Footer saat ini global. Beberapa layar (scoring, search) butuh petunjuk spesifik
yang lebih jelas.
- **Valid:** SEBAGIAN — footer sudah ada, tapi isinya kadang terlalu umum.
- **Tindakan:** Pastikan tiap layar memanggil `tui_footer()` dengan teks konteks
  (sudah dilakukan di registration/scoring; cek ranking/recap sudah memanggilnya).

## 3.3 Indikator "wajib diisi" pada input nama
Saat pendaftaran, user tidak tahu berapa minimal peserta sampai error muncul.
- **Valid:** YA.
- **Tindakan:** Di layar pendaftaran, tampilkan "Minimal 5, maksimal 7 peserta"
  secara permanen (bukan cuma saat error). Progress bar kuota sudah ada, tinggal
  tambahkan label "minimal 5" di dekatnya.

## 3.4 Shortcut keyboard di menu (angka 1-5 langsung pilih)
Selain panah, izinkan tekan angka untuk loncat ke menu tersebut.
- **Valid:** YA — mempercepat navigasi.
- **Tindakan:** Di switch menu, tambahkan case '1'..'5' yang set `selected` lalu
  jalankan aksi sama seperti ENTER.
