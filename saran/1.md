# Saran UI/UX #1 — Navigasi & Feedback Immediat

**Target:** Alur dasar menu dan input.

## 1.1 Ganti tombol panah menjadi 'v' / '^' secara konsisten
Footer menu saat ini menulis "Navigasi: [v/^]" tapi kode hanya menangani panah
(KEY_UP/KEY_DOWN). Pengguna yang membaca footer jadi bingung karena tombol 'v'
tidak berfungsi.
- **Valid:** YA — ini bug nyata. Footer dan kode tidak sinkron.
- **Tindakan:** Samakan. Pilih salah satu: (a) handle 'v'/'^' di switch menu, atau
  (b) ubah footer jadi "Navigasi: [^/v] (panah)". Opsi (b) lebih mudah & konsisten
  dengan terminal lain.

## 1.2 Konfirmasi keluar yang ramah
Saat ini ESC langsung keluar tanpa konfirmasi (di menu utama). Risiko: pengguna
tersesat menekan ESC dan langsung keluar, kehilangan sesi.
- **Valid:** SEBAGIAN — di sesi ini kita sudah rencanakan `tui_confirm`, tapi belum
  dipasang di menu utama. Ini masih pekerjaan terbuka, jadi VALID sebagai pengingat.
- **Tindakan:** Pasang `tui_confirm("Keluar dari aplikasi?")` pada ESC dan pilihan 0.

## 1.3 Pesan error langsung di baris input (tanpa ganti layar)
Di scoring, saat zona salah layar "di-clear" lalu digambar ulang baru tampil error.
Terlalu banyak kedip layar.
- **Valid:** YA — pengalaman lebih halus kalau error ditampilkan di baris feedback
  saja ( sudah ada pola `draw_scoring_screen(part, kick_num, msg, is_error)` ),
  tanpa `tui_clear()`.
- **Tindakan:** Hapus `tui_clear()` di cabang error scoring.

## 1.4 Highlight baris terpilih lebih kontras
Baris terpilih pakai `COLOR_HIGHLIGHT` (cyan terbalik). Untuk terminal 16-warna
kadang sulit dibedakan.
- **Valid:** YA (peningkatan kecil).
- **Tindakan:** Tambahkan `A_BOLD` pada `tui_highlight_row`.
