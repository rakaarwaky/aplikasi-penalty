
# 🎯 Indikator UI/UX Bagus untuk Aplikasi Console (C)

Aplikasi Anda berbasis **console (CLI)**, jadi konsep UI/UX tidak sama dengan aplikasi grafis. Tapi justru di console, UX yang baik itu **terlihat dari bagaimana user berinteraksi dengan teks**. Berikut indikator lengkapnya:

---

## ✅ 1. Navigasi & Struktur Menu

- [ ] Menu utama **bernomor** dan tersusun rapi
- [ ] Pengguna **selalu tahu posisinya** di mana (ada header/breadcrumb)
- [ ] Ada opsi `0. Kembali` atau `Back` di setiap submenu
- [ ] Flow menu **logis** dan tidak membuat user bingung harus ke mana

```
=========================================
   LOMBA TENDANGAN PENALTI - MENU UTAMA
=========================================
   1. Daftar Peserta Baru
   2. Input Hasil Tendangan
   3. Lihat Peringkat
   4. Cari Peserta
   5. Cetak Rekapitulasi
   0. Keluar
=========================================
Pilih menu [0-5]: 
```

---

## ✅ 2. Input Handling yang Ramah (PENTING!)

- [ ] **Input invalid tidak membuat program crash**
- [ ] Pesan error **jelas dan memberi tahu** cara memperbaiki
- [ ] **Validasi tipe data** (misal: harus angka, tidak boleh huruf)
- [ ] Memberi contoh format yang benar saat user salah input
- [ ] **Tidak mengulang dari awal** hanya karena 1 input salah

```c
// ❌ BURUK
scanf("%d", &pilihan);  // user ketik "abc" → crash/infinite loop

// ✅ BAIK  
printf("Input harus angka antara 1-5. Coba lagi: ");
while(scanf("%d", &pilihan) != 1 || pilihan < 1 || pilihan > 5) {
    while(getchar() != '\n'); // bersihkan buffer
    printf("Input tidak valid. Coba lagi [1-5]: ");
}
```

---

## ✅ 3. Feedback & Komunikasi dengan User

- [ ] Setiap aksi sukses → ada **pesan konfirmasi** ("✓ Peserta berhasil didaftarkan!")
- [ ] Setiap aksi gagal → ada **pesan error yang spesifik**
- [ ] Proses panjang → ada **progress indicator** (bukan blank screen)
- [ ] Tidak ada "dead silence" (layar diam tanpa kabar)

```
[████████████░░░░░░░░] 60% - Menghitung peringkat...
```

---

## ✅ 4. Formatting Output yang Rapi

- [ ] Data ditampilkan dalam **tabel yang rapi** (aligned)
- [ ] Angka, tanggal, skor punya **format yang konsisten**
- [ ] Penggunaan **garis pembatas** untuk memisahkan section
- [ ] **ANSI colors** (opsional tapi sangat membantu) untuk highlight

```
┌─────┬──────────────┬──────┬───────┬─────────┐
│ No  │ Nama Peserta │ Gol  │ Miss  │ Skor    │
├─────┼──────────────┼──────┼───────┼─────────┤
│  1  │ Budi Santoso │   4  │   1   │   12    │
│  2  │ Andi Wijaya  │   3  │   2   │    9    │
└─────┴──────────────┴──────┴───────┴─────────┘
```

---

## ✅ 5. Error Handling yang Graceful

- [ ] Program **tidak pernah crash** mendadak
- [ ] **Segfault/abort tidak terjadi** meski user coba hal aneh
- [ ] Ada **safe exit** dari kondisi error
- [ ] File I/O error ditangani (file tidak ada, tidak bisa dibuka, dll)

---

## ✅ 6. Konfirmasi Aksi Berbahaya

- [ ] Aksi seperti **delete data, keluar program, reset** → minta konfirmasi
- [ ] Default pilihan adalah yang **paling aman** (biasanya "Tidak")

```
Apakah Anda yakin ingin keluar? Data belum disimpan. (y/n): 
```

---

## ✅ 7. Fitur Memanjakan User (Nice-to-Have)

- [ ] **Clear screen** saat berpindah menu (tidak berantakan)
- [ ] **Search/Pencarian** yang toleran (case-insensitive)
- [ ] **Default values** yang masuk akal untuk input opsional
- [ ] **Tie-breaker** ditampilkan dengan jelas saat skor sama
- [ ] **Keyboard shortcuts** (misal: tekan 'q' untuk keluar dari mana saja)
- [ ] **Save/Load state** agar progress tidak hilang saat program ditutup

---

## 🧪 Checklist Singkat: Aplikasi Anda Sudah Bagus Jika...

| Aspek              | Indikator                                              |
| ------------------ | ------------------------------------------------------ |
| 🎯 Intuitif        | User baru bisa pakai**tanpa membaca manual**     |
| 🛡️ Tahan Banting | User bisa input apa saja,**program tidak crash** |
| 💬 Komunikatif     | User**selalu tahu** apa yang terjadi             |
| 🎨 Rapi            | Output**mudah dibaca** dalam sekali lihat        |
| ⚡ Efisien         | Tugas umum selesai dalam**beberapa keystroke**   |
| 🤝 Ramah           | Pesan error**membantu, bukan menyalahkan**       |

---

## 📝 Saran Praktis untuk Aplikasi Anda

Mengingat ini aplikasi **lomba tendangan penalti**, perhatikan flow khusus:

1. **Pendaftaran** → Validasi nama tidak kosong, nomor peserta unik
2. **Input tendangan** → Tampilkan siapa yang sedang menendang, beri instruksi jelas
3. **Perhitungan skor** → Tampilkan breakdown skor agar transparan
4. **Tie-breaker** → Jelaskan aturan tie-breaker saat terjadi seri
5. **Rekapitulasi** → Format rapi untuk dicetak/disimpan ke file
