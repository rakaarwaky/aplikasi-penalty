
Saya akan melakukan audit menyeluruh terhadap aplikasi ini dari segi bug dan fungsionalitas. Mari saya analisis setiap file secara sistematis.

## Temuan Bug & Masalah Fungsionalitas

### 1. **Bug Kritis: Recap Aggregate Meminjam Ranking Protocol yang Salah**

**File:** `src/recap/root_recap_container.c`

Masalah: `root_recap_build()` membuat instance `RankingProtocol` lokal yang menunjuk ke `capabilities_ranking_compute`, tetapi `RecapAggregate` seharusnya menerima pointer ke `RankingProtocol` yang sudah ada (dari `root_ranking_build()`). Ini menyebabkan:

- Dua instance `RankingProtocol` terpisah (satu di `root_ranking_build()`, satu di `root_recap_build()`)
- Tidak konsisten jika ada perubahan pada ranking container
- Memori statis yang redundan

**Solusi:** `root_recap_build()` harus menerima `RankingProtocol*` sebagai parameter, atau recap container harus di-build setelah ranking container dengan pass-by-reference.

### 2. **Bug Fungsional: Menu State Indicator Tidak Akurat**

**File:** `src/cli/surfaces_menu_command.c`

Masalah: Indikator `[SUDAH]` untuk menu 1 (Registration) muncul ketika `state != STATE_INIT`, tetapi seharusnya hanya muncul ketika `state == STATE_REGISTERED || state == STATE_COMPLETED` DAN `participant_count >= MIN_PARTICIPANTS`. Saat ini, jika user daftar 5 peserta lalu state berubah ke `STATE_REGISTERED`, indikator `[SUDAH]` muncul — ini benar. Tapi jika user baru daftar 1 peserta (belum cukup), state masih `STATE_INIT`, jadi `[BLOKIR]` muncul — ini juga benar.

Namun ada masalah lebih serius: **Menu 2 (Scoring) tidak menunjukkan `[BLOKIR]` ketika `state == STATE_INIT`**. Di `draw_menu()`, kondisi untuk menu 2:

```c
else if (i == 2 && state == STATE_INIT) strcat(label, "  [BLOKIR]");
```

Ini benar. Tapi **menu 4 (Search)** tidak punya indikator `[BLOKIR]` sama sekali! Di PRD, search hanya aktif setelah registrasi (`state != STATE_INIT`), tapi di UI tidak ada indikator status.

### 3. **Bug Kritis: Buffer Overflow Potensial di surfaces_registration_command.c**

**File:** `src/cli/surfaces_registration_command.c`

```c
char buffer[64];
// ...
mvgetnstr(row, 22, buffer, 30);
```

Masalah: `mvgetnstr` membatasi 30 karakter, tapi `ParticipantNameVO.value` hanya `MAX_NAME_LENGTH + 1 = 31` byte. Jika user memasukkan 30 karakter + null terminator = 31 byte, ini pas. Tapi validasi panjang nama di `capabilities_registration_validator.c`:

```c
if (len > (size_t)MAX_NAME_LENGTH) return REG_NAME_TOO_LONG;
```

Ini memeriksa `len > 30`, artinya 30 karakter diperbolehkan. Namun, `strncpy(name.value, buffer, MAX_NAME_LENGTH)` di `capabilities_registration_appender.c` hanya menyalin 30 karakter, dan `name.value[MAX_NAME_LENGTH] = '\0'` men-set null terminator di indeks 30 (31 byte total). Ini sebenarnya benar.

**Tapi ada masalah lain:** Di `cli_surfaces_registration_execute()`, setelah memasukkan nama, buffer di-clear dengan `memset(buffer, 0, sizeof buffer)`, tapi saat memeriksa "selesai":

```c
char lower[64];
for (i = 0; i < (int)len; i++) lower[i] = buffer[i];
lower[len] = '\0';
```

Jika `len == 63` (maksimum sebelum null terminator), `lower[63] = '\0'` aman karena `lower` size 64. Tapi `mvgetnstr` membatasi 30, jadi ini tidak akan terjadi.

### 5. **Bug Kritis: Memory Leak / Use After Free Potensial pada Static Variables**

**File:** `src/registration/root_registration_container.c`, `src/scoring/root_scoring_container.c`, dll.

```c
RegistrationAggregate root_registration_build(void) {
    static RegistrationProtocol protocol;
    // ...
    RegistrationAggregate a;
    a.protocol = &protocol;
    return a;
}
```

Masalah: `RegistrationAggregate a` adalah local variable yang di-return by value. Struct ini berisi pointer ke `static RegistrationProtocol`. Ini aman karena `protocol` static. Tapi jika `RegistrationAggregate` di-copy (pass by value ke fungsi lain), pointer masih menunjuk ke static yang sama — ini sebenarnya intended behavior.

**Namun**, ada masalah lebih serius: **Tidak ada validasi NULL pointer yang konsisten di agent layer**. Contoh di `agent_registration_orchestrator.c`:

```c
if (agg == NULL || state == NULL || name == NULL) return REG_NAME_EMPTY;
```

Ini mengembalikan `REG_NAME_EMPTY` ketika `name == NULL`, tapi juga ketika `agg == NULL` atau `state == NULL`. Error code yang sama untuk kasus berbeda tidak informatif. Seharusnya ada error code terpisah untuk NULL aggregate/pointer.

### 6. **Bug Fungsional: Tie-Breaker Ranking Tidak Sesuai PRD**

**File:** `src/ranking/capabilities_ranking_calculator.c`

```c
for (int z = MAX_ZONE; z >= 1; z--)
    if (x->zone_freq[z] != y->zone_freq[z])
        return y->zone_freq[z] - x->zone_freq[z];
```

Masalah: PRD 2.3 menyebutkan tie-breaker: zona 5 terbanyak, lalu 4, 3, 2, 1. Kode ini melakukan exactly that. Tapi **PRD tidak menyebutkan zona 0** dalam tie-breaker, dan kode juga tidak memeriksa zona 0 (loop dari 5 ke 1). Ini benar.

Tapi ada masalah: **Jika dua peserta benar-benar identik (semua zona sama), mereka dianggap seri**. Kode ini:

```c
if (compare_entries(&out[i - 1], &out[i]) == 0)
    out[i].rank = out[i - 1].rank;
else
    out[i].rank = i + 1;
```

Ini benar untuk peringkat seri. Tapi **peringkat setelah seri salah**: jika 2 peserta seri di peringkat 1, peserta ke-3 harus peringkat 3. Kode ini `out[i].rank = i + 1` → untuk i=2 (peserta ke-3), rank = 3. Benar!

### 7. **Bug Kritis: Competition State Transition Tidak Aman**

**File:** `src/registration/capabilities_registration_appender.c`

```c
if (state->state == STATE_INIT && state->participant_count >= MIN_PARTICIPANTS)
    state->state = STATE_REGISTERED;
```

Masalah: State transition dari `STATE_INIT` ke `STATE_REGISTERED` hanya terjadi saat menambah peserta ke-N (N >= 5). Tapi tidak ada mekanisme untuk **membatalkan** atau **mengurangi** peserta. Jika user salah input, tidak ada cara menghapus peserta. Ini bukan bug teknis tapi keterbatasan fungsional.

**Masalah lebih serius:** State transition dari `STATE_REGISTERED` ke `STATE_COMPLETED` ada di `capabilities_scoring_score_calculator.c`:

```c
if (state->state == STATE_REGISTERED) {
    int all = 1;
    for (int i = 0; i < state->participant_count; i++)
        if (state->participants[i].kick_count < TOTAL_KICKS) { all = 0; break; }
    if (all) state->state = STATE_COMPLETED;
}
```

Ini hanya berjalan saat `record_kick` dipanggil. Jika semua peserta selesai tendangan, state berubah. Tapi jika somehow state tidak berubah (misal race condition, tapi ini single-threaded), tidak ada fallback.

### 11. **Bug Kritis: Recap Agent Tidak Validasi Kapasitas Array**

**File:** `src/recap/agent_recap_orchestrator.c`

```c
RecapError agent_recap_prepare(RecapAggregate *agg,
                               const CompetitionState *state,
                               RankingEntryVO *ranking,
                               SearchResultVO *details,
                               int capacity) {
    if (agg == NULL || state == NULL || ranking == NULL || details == NULL) return RC_NOT_READY;
    if (agg->ranking->compute_ranking(state, ranking, capacity) != RK_OK)
        return RC_NOT_READY;
    return agg->protocol->prepare_details(state, details, capacity);
}
```

Masalah: `ranking` dan `details` adalah array terpisah. `compute_ranking` mengisi `ranking` dengan `participant_count` entries. Tapi `prepare_details` mengisi `details` dengan indeks = `participant_id`. Di `cli_surfaces_recap_command.c`:

```c
const RankingEntryVO *r = &ranking[i];
const SearchResultVO *d = &details[r->participant_id];
```

Ini mengasumsikan `details` di-index oleh `participant_id`, yang benar berdasarkan implementasi `capabilities_recap_prepare_details`. Tapi jika `ranking` array di-sort oleh `compute_ranking`, indeks `i` di ranking tidak sama dengan `participant_id`. Kode recap command menggunakan `r->participant_id` untuk mengakses `details`, yang benar.

**Tapi ada masalah:** `agent_recap_prepare` tidak memeriksa apakah `capacity >= state->participant_count` sebelum memanggil `compute_ranking`. Jika `capacity` terlalu kecil, `compute_ranking` akan return `RK_NO_PARTICIPANT`, tapi ini sudah di-handle.

### Perbaikan 4: Input Buffer Flush Setelah Error di Scoring

### Updated file: `src/cli/surfaces_scoring_command.c`

```c
#include "cli/module.cli.h"
#include "tui/infrastructure_tui_adapter.h"

#include <stdio.h>
#include <string.h>
#include <ctype.h>

static ScoringError read_zone(ZoneVO *out) {
    char buf[32];
    echo();
    curs_set(1);
    memset(buf, 0, sizeof buf);
    getnstr(buf, 10);
    curs_set(0);
    noecho();

    /* Flush sisa buffer jika ada (ncurses getnstr seharusnya handle, tapi untuk safety) */
    int i = 0;
    while (buf[i] != '\0' && buf[i] != '\n' && buf[i] != '\r') {
        if (!isdigit((unsigned char)buf[i]) && buf[i] != '-') return SC_INVALID_ZONE;
        i++;
    }
  
    /* Periksa apakah input kosong */
    if (i == 0) return SC_INVALID_ZONE;
  
    int z;
    if (sscanf(buf, "%d", &z) != 1) return SC_INVALID_ZONE;
    out->value = z;
    return SC_OK;
}
```

```c
static void draw_menu(int selected, CompetitionStateKind state) {
    tui_clear();

    /* Title */
    attron(COLOR_PAIR(COLOR_TITLE) | A_BOLD);
    tui_print_centered(1, "APLIKASI LOMBA TENDANGAN PENALTI");
    attroff(COLOR_PAIR(COLOR_TITLE) | A_BOLD);

        snprintf(label, sizeof label, "%s", menu_labels[i]);

        /* Add status indicators berdasarkan state machine PRD */
        if (i == 1) {  /* Registration */
            if (state == STATE_INIT) strcat(label, "  [AKTIF]");
            else strcat(label, "  [SUDAH]");
        }
        else if (i == 2) {  /* Scoring */
            if (state == STATE_INIT) strcat(label, "  [BLOKIR]");
            else if (state == STATE_COMPLETED) strcat(label, "  [SELESAI]");
            else strcat(label, "  [AKTIF]");
        }
        else if (i == 3) {  /* Ranking */
            if (state != STATE_COMPLETED) strcat(label, "  [BLOKIR]");
            else strcat(label, "  [AKTIF]");
        }
        else if (i == 4) {  /* Search */
            if (state == STATE_INIT) strcat(label, "  [BLOKIR]");
            else strcat(label, "  [AKTIF]");
        }
        else if (i == 5) {  /* Recap */
            if (state != STATE_COMPLETED) strcat(label, "  [BLOKIR]");
            else strcat(label, "  [AKTIF]");
        }

        if (i == selected)
            tui_highlight_row(row, BOX_START_COL, BOX_WIDTH, label);
        else
            tui_normal_row(row, BOX_START_COL, BOX_WIDTH, label);

    /* Footer */

    mvprintw(BOX_START_ROW + MENU_ITEMS + 3, BOX_START_COL + 2,
             "Navigate: v/^  Select: Enter  Quit: 0/ESC");

int cli_surfaces_menu_run(RegistrationAggregate *reg,
                          ScoringAggregate *sc, RankingAggregate *rk,
                          SearchAggregate *sr, RecapAggregate *rc,
                          CompetitionState *state) {
    int selected = 1;
    int running = 1;

    while (running) {
        draw_menu(selected, state->state);
        int key = tui_getch();

        switch (key) {
            case TUI_KEY_UP:
                selected--;
                if (selected < 0) selected = MENU_ITEMS - 1;
                break;
            case TUI_KEY_DOWN:
                selected++;
                if (selected >= MENU_ITEMS) selected = 0;
                break;
            case '0':
                if (confirm_exit()) running = 0;
                break;
            case TUI_KEY_ENTER:
                if (selected == 0) {
                    if (confirm_exit()) running = 0;
                } else if (selected == 1) {
                    if (state->state == STATE_INIT)
                        cli_surfaces_registration_execute(reg, state);
                } else if (selected == 2) {
                    if (state->state != STATE_INIT)
                        cli_surfaces_scoring_execute(sc, state);
                } else if (selected == 3) {
                    if (state->state == STATE_COMPLETED)
                        cli_surfaces_ranking_execute(rk, state);
                } else if (selected == 4) {
                    if (state->state != STATE_INIT)
                        cli_surfaces_search_execute(sr, state);
                } else if (selected == 5) {
                    if (state->state == STATE_COMPLETED)
                        cli_surfaces_recap_execute(rc, state);
                }
                break;
            case TUI_KEY_ESC:
                if (confirm_exit()) running = 0;
                break;
        }
    }
    return 0;
}
```

```c
        attroff(COLOR_PAIR(COLOR_SUCCESS));
    }

    int row = 7 + state->participant_count + 1;
    char buffer[64];
    while (state->participant_count < MAX_PARTICIPANTS) {
        attron(COLOR_PAIR(COLOR_MENU));

        size_t len = strlen(buffer);
        while (len > 0 && (buffer[len - 1] == '\n' || buffer[len - 1] == '\r'))
```
